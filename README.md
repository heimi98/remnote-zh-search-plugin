# RemNote 中文搜索增强

为 RemNote 提供更可靠的中文关键词搜索体验。

这个插件会在本地读取你当前可访问的 Rem 内容，自建一份适合中文检索的索引。相比依赖原生分词的搜索方式，它更适合直接输入中文词组、短语或句子片段进行查找。

## 适用场景

- 原生搜索对中文命中不稳定，搜得到英文，搜不准中文
- 想直接输入完整中文词语，比如“功率”“边际成本”“长期记忆”
- 需要从笔记正文里检索，而不只是文档标题
- 想在移动端也有一个稳定、直接的中文搜索入口

## 主要功能

- 桌面端可通过命令或快捷键 `mod+alt+f` 打开中文搜索弹窗
- 移动端可通过页面头部入口或侧边栏按钮打开同一搜索界面
- 首次使用时自动建立中文搜索索引，后续可复用本地缓存
- 支持匹配 Rem 正文内容
- 可选匹配 `backText`
- 提供“重建中文搜索索引”命令，方便在大量笔记更新后手动刷新
- 支持浅色 / 深色主题自动切换
- 支持桌面端与移动端

## 使用方式

### 桌面端

1. 安装插件
2. 使用快捷键 `mod+alt+f`
3. 或在 RemNote 命令菜单中搜索“打开中文搜索”
4. 输入中文关键词后，左侧查看匹配结果，右侧查看当前选中项预览

### 移动端

1. 安装插件
2. 点击页面头部的中文搜索入口
3. 或从插件侧边栏入口打开中文搜索
4. 输入关键词后查看结果列表与预览

## 搜索说明

- 插件会在本地建立索引，不会把你的笔记内容上传到第三方服务器
- 搜索结果主要基于当前插件可读取到的 Rem 内容
- 索引建立完成后，常见中文词语检索会明显比原生搜索稳定
- 如果你刚刚导入、移动或大量修改了笔记，建议手动执行一次“重建中文搜索索引”

## 设置项

插件提供以下设置：

- `检索背面内容`
说明：
开启后，搜索结果会同时匹配 Rem 的 `backText`

- `最大结果数`
说明：
控制每次搜索最多显示多少条匹配结果

- `启动时重建索引`
说明：
开启后，每次插件激活都会进行一次全量索引重建

## 权限说明

当前插件只请求 RemNote 的只读权限：

- `All / Read`

这些权限仅用于读取可访问笔记并建立搜索索引，不会修改你的笔记内容。

## 兼容性

- 支持移动端安装与使用
- 支持 RemNote 浅色 / 深色主题
- 索引优先保存在本地缓存；如果本地缓存不可用，会退回会话缓存或内存缓存

## 常见问题

### 为什么第一次打开会稍慢？

因为插件需要读取当前可访问的 Rem 并建立首个中文索引。首次建立完成后，后续打开通常会快很多。

### 为什么更新笔记后搜索结果没有立刻变化？

插件会尽量复用已有索引。如果你刚做了较大规模调整，执行一次“重建中文搜索索引”即可同步最新内容。

### 这个插件会不会影响原生搜索？

不会。它提供的是独立的中文搜索弹窗，不会替换 RemNote 原有搜索功能。

## 开发

```bash
npm install
npm run dev
```

然后在 RemNote 的插件开发入口中加载：

```text
http://localhost:8080
```

## 打包

```bash
npm run build
```

构建完成后会：

- 输出发布文件到 `dist/`
- 生成可直接上传到 RemNote 官方插件平台的 `PluginZip.zip`
